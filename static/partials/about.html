About

This has been quite some time I'm running the @SublimePackages account, periodically tweeting about latest and cool packages for the Sublime Text editor. I've been asked many times to build a dedicated website listing the packages I'm tweeting about, hence this very webapp.

How does it work?

The app backend is regularly fetching packages information using the Package Control repository JSON database on Github, and stores the data in JSON files on the server filesystem — one single JSON file per package repository.

When the WebServer starts, it loads the whole database into memory so there's no file lookup when users request pages.

The memory footprint of this storage strategy remains relatively low, because there are not so many packages out there (well, if a bit more than a thousand can be considered not so many). Serving data directly from memory has proven to be one of the fastest possible way ever.

The whole application, including the HTML/CSS stuff, has been developped in 4 full days of work. It's been pretty much a fun ride.

Of course, there are many possible evolutions to be expected, but I really wanted a Viable Product as Minimal as possible :)

For suggestions or bug reports, feel free to file an issue.

Limitations

Right now, only packages hosted on Github are listed. I'll try to find time for adding support for other repository sources and Package Control JSON channel description.

Technical stack

This website has been crafted using JavaScript and the following technologies:

Backend

NodeJS, using the following packages from NPM:

  • express for setting up a very basic HTTP application server
  • async to deal with asynchronous operations
  • request to perform outgoing HTTP requests
  • github to consume the Github API
  • search-index to build a custom searchable index used by the search engine

Frontend

  • AngularJS to turn this very HTML page into a fully dynamic one
  • Twitter Bootstrap 3 to give design some love
  • Marked to transform github's README contents into HTML (this operation is performed client-side)
  • HighlightJS to bring syntax coloration for code in READMEs

But it would be probably more efficient to check the code out by yourself as the code behind this project is fully open source (MIT licensed).
